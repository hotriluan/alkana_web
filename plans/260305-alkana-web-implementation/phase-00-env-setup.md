## Requirements

### Functional
- Local WordPress dev environment running with identical PHP version to Mat Bao.
- Vite + Tailwind build compiles and outputs to `wp-content/themes/alkana/assets/`.
- Git repo initialized with proper `.gitignore` (exclude `dist/`, `node_modules/`, `.env`).
- Staging WordPress installed on Mat Bao pointing to `staging.alkana.vn` subdomain.
- SSH or SFTP access confirmed to Mat Bao cPanel.

### Non-Functional
- All team members have access to shared Git repo.
- `.env.example` documented for DB credentials.
- PHP version on Mat Bao confirmed (target: 8.1 or 8.2).
- MySQL version on Mat Bao confirmed (target: 8.0 for JSON column support; fallback schema if 5.7).

## Architecture — Local Dev Stack


## Implementation Steps

1. **Confirm Mat Bao specs**
   - Login cPanel → PHP version, MySQL version, LiteSpeed status, ImageMagick capabilities.
   - Enable SSH access (or confirm SFTP credentials).

2. **Setup local environment**
   ```bash
   # Option: Laragon (Windows) or Local by WP Engine
   # Install WordPress 6.x locally
   # Import clean DB dump from Mat Bao staging
   ```

3. **Create Git repository**
   ```bash
   git init alkana_web
   cd alkana_web
   # .gitignore: node_modules/, dist/, .env, *.log, wp-config.php
   git remote add origin <repo-url>
   ```

4. **Initialize WordPress theme directory**
   ```
   wp-content/themes/alkana/
   ├── style.css          (theme header only)
   ├── functions.php
   ├── index.php
   ├── package.json
   ├── vite.config.js
   ├── tailwind.config.js
   ├── src/
   │   ├── styles/app.css
   │   └── scripts/app.js
   └── dist/              (gitignored — generated by vite build)
   ```

5. **Configure Vite**
   ```js
   // vite.config.js
   import { defineConfig } from 'vite';
   export default defineConfig({
     build: {
       outDir: 'dist',
       rollupOptions: {
         input: { app: 'src/scripts/app.js' }
       }
     }
   });
   ```

6. **Configure Tailwind**
   ```bash
   npm install -D tailwindcss @tailwindcss/vite
   npx tailwindcss init
   # tailwind.config.js: content: ['./**/*.php']
   ```

7. **Verify build pipeline works**
   ```bash
   npm run build
   # dist/app.[hash].css + dist/app.[hash].js must appear
   ```

8. **Setup staging on Mat Bao**
   - Create subdomain `staging.alkana.vn` in cPanel.
   - Install WordPress via cPanel Softaculous.
   - Install required plugins: ACF Pro, WP Mail SMTP (for contact forms), Query Monitor (debug only).

9. **Deploy initial theme to staging**
   ```bash
   rsync -avz wp-content/themes/alkana/ user@matbao:/public_html/staging/wp-content/themes/alkana/
   ```

10. **Document credentials** in shared password manager (NOT in git).

## Risk Assessment
| Risk | Probability | Mitigation |
|---|---|---|
| Mat Bao SSH access denied | Medium | Use SFTP + rsync over password auth |
| MySQL 5.7 instead of 8.0 | Medium | Use `TEXT` instead of `JSON` for surface_types column |
| ACF Pro license not available | Low | ACF Free + custom code for repeater fallback |

## Security Considerations
- Never commit `wp-config.php`, `.env`, or any credentials to Git
- Use `.gitignore` for `wp-config.php`
- Staging site: password-protect with `.htpasswd` to prevent Google indexing
